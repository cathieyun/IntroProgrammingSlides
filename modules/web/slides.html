Common questions: where is the user input?

  • In Javascript, user input is usually fetched from the front-end.
  • I will demo how to do this in the how-a-website-works demo.

Common questions: how do functions get their variables?

  • Function inputs: the stuff between the ()
  • Function variable are created that only exist within the {} of the function.
  • Function variables' values are assigned from the function inputs.

Login Review

  • Make the register and login functions work.
  • Register: add keys and values to a dictionary.
  • Login: check values against stored values in a dictionary.
  • Understand the difference between print and return.

Websites: Back-end

  • DEMO

Websites: Code Injection attack

  • Goal: log in to the website as a user that is not you.
  • Limitation: you don't know other peoples' passwords.
  • Advantage: you can create messages to send to the server.
  • Knowledge: {$gt: ""} will give you the first thing in the database.
  • Weakness: inputs are not sanitized on this website!

Websites: Cross-site-scripting (XSS)

  • Remember this comic?
  • Inject SQL code directly into database through user input

Websites: Many many more attacks!

  • In the demo code you can also do a CSRF attack.
  • CSRF = cross-site request forgery. Another site pretends to be the victim site, uses the victim site's credentials.
  • Hackers are always coming up with more ways to get your information.
  • Security experts are always coming up with ways to make computers more secure.
